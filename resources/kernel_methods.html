Kernel methods

  • Idea: Use large (possibly infinite) set of fixed non-linear basis functions
  • Normally, complexity depends on number of basis functions, but by a "dual trick", complexity depends on the amount of data.
  • Examples:
    • Gaussian Processes
    • SVM
    • Kernel Perceptron
    • Kernel PCA

Kernel Function

  • Let ϕ(x)\phi(x)ϕ(x) be a set of basis functions that map inputs xxx to a feature space.
  • In many algorithms, this feature space only appears in the dot product ϕ(x)Tϕ(x′)\phi(x)^T\phi(x')ϕ(x)Tϕ(x′) of input pairs x,x′x, x'x,x′.
  • Define the kernel function k(x,x′)=ϕ(x)Tϕ(x′)k(x, x') = \phi(x)^T\phi(x')k(x,x′)=ϕ(x)Tϕ(x′) to be dot product of an pair x,x′x, x'x,x′ in the new space.
    • We only need to know k(x,x′)k(x, x')k(x,x′), not ϕ(x)\phi(x)ϕ(x).
  • Let Φ(x)\Phi(x)Φ(x) = [ϕ(x1),ϕ(x2),...,ϕ(xN)][\phi(x_1), \phi(x_2), ..., \phi(x_N)][ϕ(x1​),ϕ(x2​),...,ϕ(xN​)].
  • Let K=Φ(x)TΦ(x)K = \Phi(x)^T \Phi(x)K=Φ(x)TΦ(x) be the Gram matrix. It is essentially the matrix of pairwise dot products of data points in the transformed feature space defined by the kernel function.

  • This is basically called the kernel trick, where computations in the original space implicity correspond to operations in a higher-dimensional space.

  • It’s called a higher dimension because the kernel function implicitly maps the input data points from their original space (input space) to a new space (feature space) that has more dimensions than the original. Gaussian process

Formally, A Gaussian process is defined as a collection of random variables, any finite number of which have a joint Gaussian distribution. If a function f(x)f(x)f(x) is a Gaussian process, with mean function m(x)m(x)m(x) and covariance function or kernel k(x,x′),f(x)∼GP(m,k)k(x, x'), f(x) \sim \mathcal{GP}(m, k)k(x,x′),f(x)∼GP(m,k), then any collection of function values queried at any collection of input points xxx has a joint multivariate Gaussian distribution with mean vector μ\muμ and covariance matrix K:f(x1),...,f(xn)∼N(μ,K)K: f(x_1), ..., f(x_n) \sim \mathcal{N}(\mu, K)K:f(x1​),...,f(xn​)∼N(μ,K), where ui=E[f(xi)]=m(xi)u_i = E[f(x_i)] = m(x_i)ui​=E[f(xi​)]=m(xi​) and Kij=Cov(f(xi),f(xj))=k(xi,xj)K_{ij} = \mathrm{Cov}(f(x_i), f(x_j)) = k(x_i, x_j)Kij​=Cov(f(xi​),f(xj​))=k(xi​,xj​).

Informally, A gaussian process (GP\mathcal{GP}GP) is a way to model a distribution over functions. For any input point xxx, the function value f(x)f(x)f(x) is a random variable following a Gaussian distribution. For any set of input points x1,x2,...,xnx_1, x_2, ..., x_nx1​,x2​,...,xn​, the corresponding function values f(x1),f(x2),...,f(xn)f(x_1), f(x_2), ..., f(x_n)f(x1​),f(x2​),...,f(xn​) joinlty follow a multivariate Gaussian distribution, with a mean vector μ\muμ and a covariance matrix KKK determined by the mean function m(x)m(x)m(x) and covariance function k(x,x′)k(x, x')k(x,x′).

So, Is multivariate Gaussian distribution a Gaussian process then?

Well, No. A gaussian process is a 'generalization' of the Multivariate Gaussian Distribution (MGD). In MGD the number of random variables is finite. In other words, fixed number of xxx's. However, in GP\mathcal{GP}GP, the number of random variables are infinite. It models a function f(x)f(x)f(x), where xxx can take on any value in a continuous input space (infinite possibilites).

In other words, in MGD, the random variables are the finite xxx's themselves. We deal with finite-dimensional vector of random variables [x1,x2,...,xn].[x_1, x_2, ..., x_n].[x1​,x2​,...,xn​]. Whereas, in GP\mathcal{GP}GP, the random variables are the function values f(x)f(x)f(x), not xxx. For any given xxx, f(x)f(x)f(x) is a random variable, and the GP\mathcal{GP}GP specifies the joint distribution of f(x)f(x)f(x) values over all xxx. The GP\mathcal{GP}GP is trying to model any function that could fit the observed data, and for that, it assumes that there are infinitely many possible input points where the function could be evaluated. So, even though we only have a finite set of data points (in practice), the GP imagines that the function exists for any xxx in the input space. Now, if we limit ourselves to a finite number of x's, we end up with finite f(x)′sf(x)'sf(x)′s each with a Gaussian distribution and jointly multivariate gaussian distribution.

When we say that a distribution is closed under addition, we are referring to a property of a set, function, or distribution where the addition of any two elements within that set (or distribution) results in another element that is still within the same set (or distribution).

In short, f(x)f(x)f(x) is a random function, or a distribution over functions. We can gain some insights into this distribution by repeatedly sampling values for w0,w1w_0, w_1w0​,w1​, and visualizing the corresponding functions f(x)f(x)f(x), which are straight lines with slopes and different intercepts. This shows how a distribution over parameters in a model induces a distribution over function. While we often have ideas about the function we want to model −-− whether they're smooth, periodic, etc. −-− it is relatively tedious to reason about the parameters, which are largely uninterpretable. Fortunately, Gaussian processes provide an easy mechanism to reason directly about functions. Since, a Gaussian distribution is entirely defined by its first two moments, its mean and covariance matrix, a Gaussian process by extension is defined by its mean function and covariance function.

In MGD, we deal with a finite number of random variables, so the covariance matrix KKK is explicitly defined. If we have nnn random variables x1,x2,...,xnx_1, x_2, ..., x_nx1​,x2​,...,xn​, we construct n×nn\times nn×n covariance KKK, where each entry kij=Cov(xi,xj)k_{ij} = Cov(x_i, x_j)kij​=Cov(xi​,xj​). Well, in GP\mathcal{GP}GP, there are infinite xxx and thus infinite f(x)f(x)f(x) random variables. So, how do we define covariance matrix for the GP\mathcal{GP}GP then?

In GP\mathcal{GP}GP, we are working with infinite number of random variables because the input space xxx is continuo us. Instead of a fixed covariance matrix, we define a covariance matrix, also called a kernel function k(x,x′)k(x, x')k(x,x′), which specifies the covariance between the random variables f(x)f(x)f(x) and f(x′)f(x')f(x′) for an pair of input xxx and x′x'x′. While the GP\mathcal{GP}GP is defined over an infinite xxx, in practice, we only evaluate the GP\mathcal{GP}GP on a finite subset of points (e.g., training data or query points).

For nnn input points x1,x2,...,xnx_1, x_2, ..., x_nx1​,x2​,...,xn​, the kernel function k(x,x′)k(x, x')k(x,x′) is evaluated pairwise for all points to produce a covariance matrix KKK, where Kij=k(xi,ij)K_{ij} = k(x_i, i_j)Kij​=k(xi​,ij​). This KKK is a finite n×nn\times nn×n matrix and must be positive semi-definite (PSD) to ensure the joint multivariate Gaussian distribution is valid.

To summarize, a GP\mathcal{GP}GP simply says that any collection of function values f(x1),f(x2),...,f(xn)f(x_1), f(x_2), ..., f(x_n)f(x1​),f(x2​),...,f(xn​), indexed by any collection of inputs x1,...,xnx_1, ..., x_nx1​,...,xn​ has a joint multivariate Gaussian distribution. RBF Kernel

The properties of the Gaussian process that we used to fit the data are strongly controlled by what's called a covariance function, also known as a kernel. The Radial Basis Function (RBF, a.k.a. squared exponential, Gaussian), which has the form

kRBF(x,x′)=Cov(f(x),f(x′))=a2exp⁡(−12l2∣∣x−x′∣∣2)\begin{align*} k_{RBF}(x, x') &= \mathrm{Cov}(f(x), f(x'))\\ &= a^2\exp\left(-\frac{1}{2l^2}||x-x'||^2\right) \end{align*} kRBF​(x,x′)​=Cov(f(x),f(x′))=a2exp(−2l21​∣∣x−x′∣∣2)​

The hyperparameters are interpretable. The amplitude parameter aaa controls the vertical scale over which the function is varying, and the length-scale parameter lll controls the rate of variation (the wiggliness) of the function. Larger aaa means larger function values, and larger lll means more slowly varying functions.

The length-scale has a particularly pronounced effct on the predictions and uncertainty of the GP\mathcal{GP}GP. At ∣∣x−x′∣∣=l||x-x'||=l∣∣x−x′∣∣=l, the covariance between a pair of function values is a2e−0.5a^2e^{-0.5}a2e−0.5. At larger distance than lll i.e, ∣∣x−x′∣∣>l||x-x'|| \gt l∣∣x−x′∣∣>l, the function value diminshes implying if we want to make a prediction at a point xxx, then function values with inputs xxx such that ∣∣x−x′∣∣>l||x-x'||\gt l∣∣x−x′∣∣>l will not have a strong effect on our predictions.

It should be noted that as the length-scale increases the 'wiggliness' of the function decrease, and our uncertainty decreases. If the length-scale is small, the uncertainty will quickly increase as we move away from the data, as the datapoints will be less informative about the function values.

Well, I have already shown above why RBF intrinsically maps the input to infinite dimension but let's see it again but this time starting from the weight space. We will see how we can recover this kernel by considering a linear model with infinite feature expansions. Let's consider the linear function.

Now, let's think for a moment. The ϕ(x)\phi(x)ϕ(x) we used as a basis function was of infinite dimension, and hence the parameter vector w\mathrm{w}w should of infinite dimension as well. By moving into the function space representation, we have derived how to represent a model with an infinite number of parameters, using a finite amount of computation. A gaussian process with an RBF kernel is a universal approximator, capable of representing any continuous function.

So a Gaussian process with an RBF kernel is a model with an infinite number of parameters and has much more flexibility than any finite neural network.

Gaussian Process Regression

In a Gaussian process regression, we model the outputs y(x)y(x)y(x) as noisy observations of an underlying noise-free function f(x)f(x)f(x):

y(x)=f(x)+ϵ(x)y(x) = f(x) + \epsilon(x) y(x)=f(x)+ϵ(x)

  • f(x)f(x)f(x): The true (latent) function we aim to model.
  • ϵ(x)∼N(0,σ2)\epsilon(x) \sim \mathcal{N}(0, \sigma^2)ϵ(x)∼N(0,σ2): Additive Gaussian noise with variance σ2\sigma^2σ2.

For simplicity, we often work with:

  • Training inputs X={x1,x2,...,xn}X = \{x_1, x_2, ... , x_n\}X={x1​,x2​,...,xn​}
  • Training observations y={y(x1),y(x2),...,y(xn)}y = \{y(x_1), y(x_2), ... , y(x_n)\}y={y(x1​),y(x2​),...,y(xn​)}

And, we want to predict the outputs for new test inputs X∗={x∗1,x∗2,...,x∗m}X_\ast=\{x_\ast^1, x_\ast^2,..., x_\ast^m\}X∗​={x∗1​,x∗2​,...,x∗m​}

At the training points XXX, the vector of latent function values f=[f(x1)f(x2)...f(xn)]f=\begin{bmatrix} f(x_1) \\ f(x_2) \\ ... \\ f(x_n) \end{bmatrix}f=​f(x1​)f(x2​)...f(xn​)​​ follows:

Where:

  • K(X,X)K(X, X)K(X,X) is the covariance matrix of size n×nn\times nn×n, constructed using the kernel k(xi,xj)k(x_i, x_j)k(xi​,xj​).

The log marginal likelihood appears in Gaussian Process Regression (GPR) when learning the kernel hyperparameters. While the predictive distribution describes the posterior for predictions at test points, the log marginal likelihood is used to evaluate how well the model explains the observed training data.

  • Role of Log Marginal Likelihood

    The log marginal likelihood log⁡p(y∣X,θ)\log p(y|X, \theta)logp(y∣X,θ), quantifies how probable the observed data yyy is under the Gaussian Process model given:

    • Training inputs XXX
    • The kernel kθk_\thetakθ​ parameterized by hyperparameters θ\thetaθ, and
    • Noise variance σ2\sigma^2σ2

    We use the log marginal likelihood to learn the best hyperparameters θ\thetaθ (e.g., kernel parameters like length scale, variance, or noise level) by maximizing this value.

  • Connection to Predictive Distribution

    Before we predict f∗f_\astf∗​ for test points X∗X_\astX∗​, we need the best kernel hyperparameters θ\thetaθ. These are obtained by optimizing the log marginal likelihood.

    log⁡p(y∣X,θ)=−12yT(Kθ(X,X)+σ2I)−1y−12log∣Kθ(X,X)+σ2I∣−n2log⁡2π\log p(y|X, \theta) = -\frac{1}{2}y^T (K_\theta(X, X)+\sigma^2I)^{-1}y-\frac{1}{2}log|K_\theta(X, X)+\sigma^2I| - \frac{n}{2}\log 2\pi logp(y∣X,θ)=−21​yT(Kθ​(X,X)+σ2I)−1y−21​log∣Kθ​(X,X)+σ2I∣−2n​log2π

    The log marginal likelihood has three terms:

    1. Model fit term yT(Kθ(X,X)+σ2I)−1yy^T (K_\theta(X, X)+\sigma^2I)^{-1}yyT(Kθ​(X,X)+σ2I)−1y:
    • Measures how well the model explains the training data
    • Encourages hyperparameters that make the data likely under the GP\mathcal{GP}GP prior.
    1. Model complexity term log∣Kθ(X,X)+σ2I∣log|K_\theta(X, X)+\sigma^2I|log∣Kθ​(X,X)+σ2I∣:
    • Penalizes models that are too flexible (complex)
    • Encourages hyperparameters that make the kernel matrix well-conditioned.
    • A large determinant indicates a high-dimensional "effective volume" of the function space, meaning the model is highly complex.

  • Why is Log Marginal Likelihood important?

    • Hyperparameter Optimization: To predict f∗f_\astf∗​, the kernel k0(x,x′)k_0(x, x')k0​(x,x′) must be well-tuned to the data. We achieve this by maximizing log⁡p(y∣X,θ)\log p(y|X, \theta)logp(y∣X,θ), finding θ∗\theta_\astθ∗​ that balances model fit and complexity.
    • Link to Predictions: Once θ∗\theta_\astθ∗​ is learned, it is used to compute the predictive mean μ∗\mu_\astμ∗​ and covariance Σ∗\Sigma_\astΣ∗​.
